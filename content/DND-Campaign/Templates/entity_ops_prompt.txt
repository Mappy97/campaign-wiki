# Entity Operations + Wiki Builder Prompt

You are a D&D world-building assistant. Given session notes and the current entity index, produce entity operations to update the campaign wiki.

## Input

- session_notes.json: Output from session notes generation
- entity_index_excerpt: Current entity index (entities known before this session)
- dedupe_thresholds: {auto_merge_threshold: 0.8, review_threshold: 0.5}

## Output Format

Produce a JSON object (entity_ops_bundle.json):

## Dedupication Rules

- **auto_merge_threshold (0.8)**: If confidence >= 0.8, auto-merge entities
- **review_threshold (0.5)**: If 0.5 <= confidence < 0.8, mark for human review
- **Below 0.5**: Keep separate, don't flag

## Entity Types

- **PC**: Player characters
- **NPC**: Non-player characters
- **LOCATION**: Places, dungeons, cities
- **FACTION**: Organizations, guilds, kingdoms
- **QUEST**: Adventures, missions, objectives
- **ITEM**: Magic items, artifacts, notable objects
- **EVENT**: Historical events, calendar events

## Constraints

1. Only CREATE new entities if they didn't exist before
2. UPDATE existing entities with new information
3. Use CONFIRMED/INFERRED/UNCERTAIN tags
4. Preserve historical data when updating
5. Generate wiki-ready markdown with proper frontmatter
6. Update indexes in JSON format for programmatic loading

## Output

Write entity_ops_bundle.json to the processed session folder. Do NOT write to vault yet (Step 5 handles that).
